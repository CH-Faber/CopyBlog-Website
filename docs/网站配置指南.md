# VermilionVoid 博客配置指南

本文档详细说明如何自定义和配置你的 VermilionVoid 博客。所有可修改的内容按区域分类整理，方便快速定位。

---

## 1. 全局站点配置

### 1.1 站点 URL 与构建设置

**文件：** `astro.config.mjs`

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| `site` | `"https://faberhu.top"` | 站点正式域名，影响 sitemap、RSS 和 OG 标签 |
| `output` | `"static"` | 构建模式（静态生成） |
| `trailingSlash` | `"always"` | URL 尾部斜杠策略 |

### 1.2 SEO 元信息

**文件：** `src/layouts/BaseLayout.astro`（第 19–20 行）

```
const defaultTitle = "My Blog | Astro Template"
const defaultDescription = "探索金融、社会与人工智能的交汇点"
```

- `defaultTitle`：浏览器标签页默认标题，也用于 OG 和 Twitter 卡片
- `defaultDescription`：站点默认描述，用于搜索引擎
- 第 57 行 `og:site_name`：Open Graph 站点名称

**文件：** `src/pages/index.astro`（第 11–12 行）

```
const title = "My Blog | Astro Template"
const description = "A modern, fast, and feature-rich Astro blog template."
```

首页独立的标题和描述，会覆盖 BaseLayout 的默认值。

**文件：** `src/pages/about.astro`（第 8–9 行）

```
const title = "About | My Blog"
const description = "Learn more about the author of this blog."
```

关于页的标题和描述。

### 1.3 分页设置

**文件：** `src/lib/constants.ts`

```
export const PAGE_SIZE = 10
```

控制首页和归档页每页显示的文章数量。

### 1.4 统计分析

**文件：** `src/layouts/BaseLayout.astro`（第 99–134 行）

- **Google Analytics**：第 107 行的 `G-1DF2LNJPSY` 替换为你的 GA4 跟踪 ID
- **Microsoft Clarity**：第 119 行的 `rkga1fxqqs` 替换为你的 Clarity 项目 ID

## 2. 个人资料

**文件：** `src/data/profile.ts`

```typescript
export const profile = {
  name: "John Doe",                              // 显示名称
  bio: "A template for awesome Astro blogs.",    // 个人简介
  avatar: "/avatar.webp",                        // 头像图片路径（相对于 public/）
  links: [                                       // 社交链接
    { type: "github",  name: "GitHub",  url: "https://github.com/yourusername" },
    { type: "twitter", name: "Twitter", url: "https://twitter.com/yourusername" },
    { type: "email",   name: "Email",   url: "mailto:hello@example.com" },
  ],
}
```

**头像图片**放在 `public/` 目录下，路径对应 `avatar` 字段。

**支持的社交链接类型：** `"qq"` | `"music"` | `"github"` | `"twitter"` | `"email"`

## 3. 导航栏

### 3.1 顶部导航菜单

```javascript
const navItems = [
  { name: "首页",   href: "/#home-main" },
  { name: "时间线", href: "/timeline/" },
  { name: "偶得",   href: "/thoughts/" },
```

## 6. 内容管理

### 6.1 文章（Posts）

**目录：** `src/content/posts/`

文章使用 Markdown 格式，frontmatter 字段如下：

```yaml
---
title: "文章标题"
published: 2026-02-28T23:23:08.000Z    # 发布时间
updated: 2026-03-01T00:00:00.000Z      # 更新时间（可选）
draft: false                            # 是否为草稿
description: "文章摘要"                  # SEO 描述
image: "/images/cover.jpg"              # 封面图（可选）
tags: ["标签1", "标签2"]                 # 标签数组
category: "分类名"                       # 分类
lang: ""                                # 语言（可选）
pinned: false                           # 是否置顶
encrypted: false                        # 是否加密
password: ""                            # 加密密码
disclaimer: "暴论"                       # 声明类型（可选，见 6.4）
---
```

### 6.2 偶得（Thoughts）

**目录：** `src/content/thoughts/`

短文/随笔格式：

```yaml
---
title: "标题"                            # 可选
published: 2026-03-01T00:00:00.000Z
tags: ["标签"]
---
```

### 6.3 文章（Articles）

**目录：** `src/content/articles/`

旧版内容集合，schema 字段：`title`, `excerpt`, `category`, `categoryLabel`, `tag`, `date`, `wordCount`, `readTime`, `image`, `pinned`

### 6.4 文章声明（Disclaimers）

**文件：** `src/data/disclaimers.ts`

内置三种声明类型：

| 类型 | 说明 |
|------|------|
| `暴论` | 争议性观点提醒 |
| `免责声明` | 通用免责声明 |
| `过时内容` | 内容时效性提醒 |

在文章 frontmatter 中通过 `disclaimer: "暴论"` 启用，支持数组形式使用多个声明。

## 7. 友链

**文件：** `src/data/friends.json`

```json
[
  {
    "categoryTitle": "Friends",
    "categoryDesc": "Links to my awesome friends.",
    "links": [
      {
        "title": "友链名称",
        "imgurl": "https://example.com/avatar.png",
        "desc": "友链描述",
        "siteurl": "https://example.com"
      }
    ]
  }
]
```

支持多个分组（每个对象是一个分类），每个分类下可以有多个友链。

## 9. 图标与 Favicon

**文件：** `src/layouts/BaseLayout.astro`（第 75–78 行）

```html
<link rel="icon" href="/icon-light-32x32.png" media="(prefers-color-scheme: light)" />
<link rel="icon" href="/icon-dark-32x32.png" media="(prefers-color-scheme: dark)" />
<link rel="icon" href="/icon.svg" type="image/svg+xml" />
<link rel="apple-touch-icon" href="/apple-icon.png" />
```

需要在 `public/` 目录下放置以下图标文件：

| 文件 | 用途 |
|------|------|
| `icon-light-32x32.png` | 浅色模式 favicon |
| `icon-dark-32x32.png` | 深色模式 favicon |
| `icon.svg` | SVG 格式 favicon |
| `apple-icon.png` | Apple 触控图标 |
| `avatar.webp` | 个人头像（由 `profile.ts` 引用） |

## 10. 快速自定义清单

如果你刚克隆了模板，按以下顺序修改即可完成个性化：

1. **`src/data/profile.ts`** — 填写你的名字、简介、头像、社交链接
2. **`public/`** — 放入你的头像和 favicon 图片
3. **`src/components/hero.tsx`** — 修改首页打字机文字、名言、按钮文字
4. **`src/components/site-header.astro`** — 修改导航栏标题和菜单项
5. **`src/components/footer.tsx`** — 修改页脚品牌、版权、备案信息
6. **`src/layouts/BaseLayout.astro`** — 修改 SEO 标题/描述、统计分析 ID
7. **`src/pages/index.astro`** — 修改首页 SEO 标题/描述
8. **`astro.config.mjs`** — 修改站点域名
9. **`src/pages/rss.xml.ts`** — 修改 RSS 标题/描述
10. **`src/data/friends.json`** — 添加你的友链
11. **`src/content/posts/`** — 开始写博客！
